Development

In this assignment, students were tasked with developing an interactive webpage in React, using interface components and tying the components to an internal data state. There were loose requirements regarding design choices and themes, but the functionality had to consist of:

  • At least two filtering categories
  • At least one sorting feature
  • An aggregator section displaying:
    • Items added to the aggregator
    • Aggregation of a property
  • At least 12 item cards, each displaying:
    • Item image
    • Categories that the filter uses
    • Field(s) that sort uses
    • Property to aggregate
    • A button that adds and removes from the aggregator

The application should also be able to handle any combination of filters, and all filters and sorting features should work together.

Part 1: Brainstorming

I began this project by first brainstorming ideas for a theme or topic. Because I enjoy traveling and exploring new places, I decided to make a travel planner application. The goal of the application is ease the stress that may come with planning a trip. The process of comparing prices, finding deals, and calculating best value destinations can be a struggle so this application aims to help users make simpler decisions.

Each item in my application will be displayed as travel destination. Each travel destination will have a property for name, package, price, people, contient, and image.

My travel planner application will have the following features:

  • Sort By:
    • Price
    • Number of guests that can be accomodated
  • Filter By:
    • Region (Europe, Asia, South America, North America)
    • Package deals (All-Inclusive, Tours and Activities Bundle, Timeshare Promotion)
  • Total (Wishlist):
    • Add to wishlist
    • Remove from wishlist
    • A display of the total price of all the destinations in the wishlist

Part 2: Coding in React

After coming up with a thorough basis for my application, I began coding it in React. As I was coding the application, I had to consider usabiity principles, and the organization of components to allow for smooth user experience.

Learnability

To address learnability, I decided to present the sorting and filtering features as radio buttons so that users could clearly see the options that they can choose from. I also decided to include a reset button instead of allowing users to click and unclick the radio buttons to go back to reset the items because I thought it would showcase the reset feature more clearly. The add and remove buttons are separated for a similar reason, so that users can clearly differentiate between the two.

Memorability

There are no intermediary steps that users have to remember in order to achieve the effect they want, making memorability very easy. If I had chosen to include dropdown menus instead of radio buttons, I think it would've been more confusing to see what filtering/sorting types align with what heading (the text that displays over the collapsed dropdown menu). Onces users know what each button does, it's easy to remember the functionality.

Efficiency

Similar to above, since there are no intermediary steps that users have to take and all buttons are clearly outlined in the UI for users to use directly, the website should be intuitive to use.

Part 3: Conclusion and Takeaways

This was the first project I had ever coded in React for so it was a very experimental journey. In the beginning I encountered conceptual difficulties, but as I made mistakes along the way I also learned more about how this Javascript library works. Through trial and error, I learned:

  • The relationship between props, components, and state
  • How state updates with user interaction
  • How to call and render an API

After completing this project, I feel as though I have a proficient grasp on the fundamentals of React. Prior to this, I had never been able to make webpages interactive, only responsive, so it was cool to be able to work with an internal data source and extract information in various ways. I'm interested to learn more about web servers and API requests as I continue to grow as a developer.
